EviLogin

EviLogin is a core functionality which secures the application from unauthorized access. Security of the Evidencer is maintained by eviLogin, which provides unique user name and password for every user.



EviManager

EviManager act as an interface between the client and the server component. It includes authenticating and authorizing the client request and incorporate auditing feature, and handle the exceptions raised in the application and provide a proper error or warning message to the user.



EviHandler

EviHandler handles any application exception or system exception raised and provides a properly formatted message to the user and maintains a log of the application exceptions.



EviCrypto

Every case hearing and police interrogation related to a crime investigation involves identities of people that need to be protected. This means any digitization or data capture applications need to provide highly reliable security features. Evidencer is powered by IBM MobileFirst for its security features.



Key Features

  • Prevents unauthorized access to the secure data
  • Provides fast encryption and decryption
  • Edited multi-media content can be copied into the attached device (DVD/USB drive) either as a secured file or an open format.
  • The user has the option to choose whether the content is to be secure or not.
  • In eviLognotes the content is encrypted and then stored in the database. On retrieval the content is decrypted and displayed in the UI


EviSign

EviSign is the security component that provides security to the digital content by applying a digital signature. Digital signatures can be used to authenticate the source of text content. The user application presents a hash code to be signed by the digital signing algorithm using a private key. The digital signature will indicate tampering if any data has been altered.



Key Features

  • Ensures security by using proprietary encryption techniques
  • Multi-level security by digitally signing the data
  • After recording, the audio video files are encrypted and then digitally signed and uploaded to the FTP location.
  • In eviPlayer the audio video file is digitally verified and decrypted before playing the content.
  • After editing a multimedia content, the file is encrypted and digitally signed, then stored in secondary storage.
