Evistore Server

Digitizing physical assets in courtroom means converting every piece of information into digital formats and ensuring these copies are stored securely. The EviStore component is the storage component in Evidencer. Since the volume of data is high, Evidencer provides options to bifurcate data into primary and secondary storage. Evidencer provides the flexibility to choose from IBM DB2, Oracle or MS SQL databases for primary storage and IBM Big Insights or Big Data Analytics (Hadoop) for secondary storage.

The use of IBM Big Insights or Big Data Analytics (Hadoop) gives the courts an edge with analytics since they can now perform analytics and generate reports on stored video data files. EviStore allows portability of data between primary and secondary storage with ease, thus giving robust storage capabilities to Evidencer. The primary storage provides security and management features that can enhance the overall user experience with Evidencer.

The EviStore component is a very crucial and important component in Evidencer that improves the speed of data analysis overall.



EviData

DB2/ Oracle or SQL Server can be used as the storage component as per the client’s requirement. Storage for Evidencer is mainly meant for storing textual information. The multimedia file is stored in an FTP server and the location of the multimedia file is stored in database. The log notes and other related textual information are also stored in the database. The application server queries are executed in the database. When a user searches for a multimedia file, the file location is retrieved from the database.



EviMedia

EviMedia is the storage component for media files. The multimedia files are stored in an FTP server. When a case file is searched, the corresponding multimedia file is retrieved from the FTP server along with other data. Efficient searching of multimedia files from an FTP location of a certain case with multiple sessions/sittings without delay is possible.
